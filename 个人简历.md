## 🚀 项目经历

### 项目一： Agent 智能助手平台（Python · LangGraph · RAG）

**项目概述**：构建面向个人场景的学习教学 & 代码智能 Agent，基于"技能优先"架构，实现 14 个活跃技能（总计 24 个可扩展储备），覆盖 RAG 全链路、知识图谱检索、智能编排、流式对话、金融投研等能力。

**技术栈**：Python 3.12 · LangChain · LangGraph · FastAPI · Streamlit · LiteLLM · ChromaDB · GraphRAG · Pydantic · SQLite · akshare

**核心工作**：

1. **技能体系设计**
    - 设计 4 层分层架构（常量层 / Pydantic 数据模型层 / 核心业务逻辑层 / 技能类层），14 个活跃技能均遵循统一规范，新增技能无需修改现有代码
    - 覆盖文本摘要、代码解释、翻译、金融投研等场景，以及 RAG 全链路（文档加载→分块→嵌入→检索→精排→问答）和 GraphRAG 知识图谱索引与检索
2. **Agent 核心设计模式落地**
    - **ReAct Agent 循环**：实现 LLM 自主 Think→Act→Observe 多轮决策循环（`run_agent`），MessageHistory 管理 `user → assistant(tool_calls) → tool` 标准角色流转，支持 max_turns 限制 + 连续失败熔断（默认 3 次）
    - **Reflection 反思模式**：基于 LangGraph StateGraph 搭建完整 4 节点闭环 —— GeneratorNode（生成初始结果）→ ExternalFeedbackNode（采集外部反馈，按技能类型分发：RAG 引用验证 / flake8 代码检查 / 通用质量评估）→ CriticNode（LLM 低温 0.1 生成结构化批评报告：severity/description/suggested_fix）→ 条件边判断（质量分数 ≥ 0.8 / 收敛检测 / 最大迭代 2 次）→ ReviserNode（基于批评意见修订，temperature=0.5）→ 回到 Critic，不达标自动终止
    - **DAG 工作流编排**：基于 LangGraph StateGraph 将知识库索引→检索等核心流水线迁移为状态图，通过配置化声明结构支持顺序边、并行 fan-out/fan-in、条件边（`build_should_continue`），SQLite Checkpoint 持久化实现断点续跑
    - **LLM 意图路由**：基于 qwen-turbo 一次调用同时完成流水线分类 + RAG 检索模式判断（vector / graphrag），取代传统关键词匹配与多次 LLM 调用。编排器 `_classify_intent_with_llm` 统一入口返回 `{"pipeline": "...", "rag_mode": "vector"|"graphrag"}`，覆盖多意图并行场景
    - **Function Calling 标准闭环**：基于 LiteLLM 统一接口调用多个大模型，完整处理 tool_calls 解析→执行→结果注入 MessageHistory→回传 LLM 的全流程
3. **RAG 能力升级**
    - 实现混合召回：稠密向量检索 + 关键词加权匹配 + 知识图谱检索
    - 集成阿里云百炼 gte-rerank 精排模型（RerankSkill）
    - 实现 QueryRewriteSkill（指代消解 / 歧义消除 / 多意图拆分）
    - 实现 LLM 意图路由：基于 qwen-turbo 轻量分类，自动判别流水线类型和 RAG 检索模式（vector / graphrag），同一入口覆盖单技能到多意图并行场景
4. **三层容错架构**
    - 第一层：Pydantic 输入输出校验
    - 第二层：指数退避自动重试装饰器
    - 第三层：熔断与降级（支持半开试探 / 自动恢复）
5. **产品化交互层**
    - 基于 FastAPI 搭建后端 API 网关（RESTful + SSE 流式输出），完整实现会话 CRUD、知识库 CRUD、规则 CRUD、LangGraph 状态查询等接口，并推送中间状态（🔍理解→📚检索→✍️生成）提升长链路场景下的用户体验
    - 使用 Streamlit 构建全功能前端界面（多会话管理、流式对话、知识库管理、规则配置、投研日报、技术雷达等 7 个页面），支持断网重连后自动拉回历史消息
    - 实现 SessionManager（会话隔离 + 持久化，复用 LangGraph Checkpoint）、RulesEngine（规则检索 + System Prompt 前缀注入，作为 LangGraph 图首个节点）、VectorStoreManager（知识库 Namespace 隔离）等产品化模块
    - 实现双检索引擎（DualRetrievalGraph）：RulesInjector→QueryRewrite→Embed→VectorRetrieve→Rerank→ContextMerge→RagAnswer 完整链路，支持 retrieval_sources 引用溯源和 applied_rules 规则可见
6. **对话记忆与长上下文管理**
    - 实现对话摘要引擎（ConversationMemory）：当会话消息 ≥ 20 条时，异步调用 LLM 生成摘要，Embedding 后存入 ChromaDB 独立集合
    - 检索阶段同时搜索知识库文档和对话历史摘要，解决长对话"早期上下文丢失"的痛点
    - 摘要检索权重可控（知识库 70%、摘要 30%），支持频率控制和旧摘要自动清理
7. **质量保障**
    - 坚持编写单元测试保证代码质量，项目累计 200+ 测试用例，核心模块达到较高覆盖率
    - 所有技能遵循统一的 4 层分层架构规范，通过 SkillManager 统一注册与调用

**项目亮点**：从底层技能架构 → Agent 核心设计模式（ReAct、Reflection、DAG）→ LLM 意图路由 → RAG 全链路 → 对话记忆引擎 → 前端交互界面，独立完成 Agent 系统从架构设计到完整实现的闭环。已实现 ReAct 多轮决策、Reflection 反思闭环、LLM 意图分类（取代关键词匹配）、LangGraph DAG 编排、Function Calling 标准闭环、双检索引擎、SSE 流式中间状态推送、对话摘要记忆等核心能力。
